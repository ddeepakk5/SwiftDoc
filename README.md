# AI-Assisted Document Authoring & Generation Platform

## ğŸ“Œ Project Overview
This project is a full-stack, AI-powered web application capable of generating, refining, and exporting professional business documents. It fulfills the assignment requirement to build a "human-in-the-loop" generation system for **Microsoft Word (.docx)** and **PowerPoint (.pptx)** formats.

The system leverages the **Google Gemini API** to generate content based on user-defined outlines, allows for granular refinement (e.g., "Make this section more formal"), and exports valid files using `python-docx` and `python-pptx`.

## ğŸš€ Key Features
* **User Authentication**: Secure JWT-based registration and login.
* **Project Management**: Dashboard to track multiple document projects.
* **Dual Format Support**:
    * **Word**: Users define a hierarchical outline (Section Headers).
    * **PowerPoint**: Users define slide counts and titles.
* **AI Generation**: Context-aware text generation for specific sections/slides.
* **Interactive Refinement**: Users can modify AI output using natural language prompts (e.g., "Shorten to 50 words").
* **Feedback System**: "Like" and "Dislike" buttons to track output quality.
* **File Export**: One-click download of the final formatted document.

## âš™ï¸ Installation & Setup Guide

### Prerequisites

  * **Python 3.9+**
  * **Node.js** & **npm**
  * **Google Gemini API Key** (Get it from [Google AI Studio](https://aistudio.google.com/))

### 1\. Backend Setup (FastAPI)

1.  Open your terminal and navigate to the `backend` folder:
    ```bash
    cd backend
    ```
2.  Create a virtual environment:
    ```bash
    # Windows
    python -m venv venv
    venv\Scripts\activate

    # Mac/Linux
    python3 -m venv venv
    source venv/bin/activate
    ```
3.  Install Python dependencies:
    ```bash
    pip install -r requirements.txt
    ```
4.  **Configure Environment Variable**:
    You must set your Gemini API key for generation to work.
    ```bash
    # Windows (CMD)
    set GEMINI_API_KEY=your_actual_api_key_here

    # Windows (PowerShell)
    $env:GEMINI_API_KEY="your_actual_api_key_here"

    # Mac/Linux
    export GEMINI_API_KEY="your_actual_api_key_here"
    ```
5.  Start the Server:
    ```bash
    uvicorn app:app --reload
    ```
    *The backend is now running at `http://localhost:8000`.*

### 2\. Frontend Setup (React)

1.  Open a **new terminal window** and navigate to the `frontend` folder:
    ```bash
    cd frontend
    ```
2.  Install Node dependencies:
    ```bash
    npm install
    ```
3.  Start the Development Server:
    ```bash
    npm run dev
    ```
    *The frontend is now running at `http://localhost:5173` (or similar, check terminal output).*

## ğŸ“– How to Use

1.  **Login**: Open the frontend URL in your browser. Register a new account and log in.
2.  **Create Project**:
      * Click **"New Project"**.
      * Select **Doc Type** (Word or PowerPoint).
      * Enter a **Topic** (e.g., "The Future of Electric Vehicles").
      * Define **Sections** (e.g., "Introduction", "Market Trends", "Conclusion").
3.  **Generate Content**:
      * You will be redirected to the **Editor**.
      * Click **"Generate"** on any section card. The AI will write content based on your topic and section title.
4.  **Refine**:
      * If you want to change the text, type an instruction in the input box (e.g., "Make this bullet points") and click **Refine**.
5.  **Export**:
      * Click the **"Download File"** button in the top right corner.
      * Open the downloaded file in Microsoft Word or PowerPoint to view the result.

## ğŸ§ª Troubleshooting

  * **Database Error**: The `ai_docs.db` file is created automatically in the `backend/` folder. If you encounter schema errors, delete `ai_docs.db` and restart the backend to recreate it.
  * **AI Errors**: If you see "Mock Mode" or error messages in the content, check your terminal. It usually means the `GEMINI_API_KEY` was not found or is invalid.
  * **CORS Issues**: Ensure the backend is running on port `8000`. The CORS configuration in `app.py` is set to allow all origins for development.
